I had the same problem. The reason is NOT a broken game.exe. Some versions of Windows are too old or might be copied. Then the MSSQL Server 2000 will be installed wrong, but the Setup tell it's complete and works. You can attach the Databases and they are working with the Server Application files, but if the Client should connect, GateServer send an request to AccountServer.exe but AccountServer.exe can't read the Database. I think this is a little mistake the programmers of Windows wanted, to break the crackers of WIndows... The MSSQL is like an "real illusion" to simulate the server... But I don't know really much about this...

My solution:
Try it on another PC or install Sun Virtualbox and install an original Windows XP on it.

Hope it will help.

Greetings from Stingray~
